## Carryover into cooldown

Items surfaced during the cycle-2 self-review (PR #7) that are
genuine work or need a design decision rather than a one-line fix.
These are the cooldown candidates; mechanical / low-risk follow-ups
landed inline in the same PR.

### High-severity (do during cooldown)

1. **Daemon payload-size ceiling.** `for line in stdin` in
   `src/ainemo/cli/daemon.py` is unbounded; a multi-GB line OOMs the
   daemon. Add a configurable max-line-bytes limit with a clear
   `ERR_INVALID_PARAMS`-style error envelope on overflow. Decide
   the default (1 MB? 10 MB?) and whether the limit is per-request
   or per-buffer.
2. **Concurrency contract sync.** `daemon.py` is single-threaded
   today but `DaemonClient.kt` uses `AtomicLong` for the
   correlation-id counter, implying future parallelism. Either lock
   `sendAndReceive` on the Kotlin side and document "single
   in-flight request only", or take the daemon multi-threaded.
   Pick one with intent.
3. **Gradle wrapper + CI workflow + composite vs standalone
   build.** The plugin currently has its own `settings.gradle.kts`
   making it the root project, but the README references
   `:gradle-plugin:build` as if it were an included build. Pick a
   layout, commit the wrapper (`gradle wrapper --gradle-version
   8.10` from a JDK 17+ host), and add a
   `.github/workflows/gradle-plugin.yml` that runs
   `./gradlew check`. The `check → functionalTest` dependency
   ([build.gradle.kts:88](../../gradle-plugin/build.gradle.kts))
   currently requires `nemo` on PATH, which breaks every
   contributor without a Python install — either detach or stub
   the daemon in CI.
4. **TM/usage `@InputFile` semantics.** `TranslateBundlesTask.kt`
   declares `tmPath` and `usageLogPath` as `@InputFile`, but the
   TM SQLite is *created* by the daemon and the usage log is an
   *output*. As inputs they break first-run (file doesn't exist)
   and incremental builds (every run mutates the TM, invalidating
   the cache for every other consumer). Needs a real
   `./gradlew check` run with the wrapper to confirm the failure
   mode and pick the right Gradle annotations
   (`@LocalState`? `@OutputFile`? drop entirely?).
5. **Cross-language nullable drift audit.** `DaemonClient.kt`
   uses `as String` casts on fields the Python side could in
   principle leave null (e.g. `target_text`, `provider`, `model`).
   Today the Python side always populates them, but there's no
   schema-level guarantee. Audit every daemon op result and
   either pin the contract via assertions or relax the Kotlin
   types to nullable. Closely related to #2 above.
6. **`DaemonClientTest` classification.** The Kotlin "unit" test
   spawns a `python3` subprocess and is therefore environment-
   dependent. Either move to `functionalTest` (depends on #3
   landing first) or gate via `Assumptions.assumeTrue(python3
   available)`. Currently silently fails on JDK-only build agents.

### Medium-severity (cooldown if room, otherwise cycle 3)

- Legacy `META-INF/gradle-plugins/...properties` descriptor
  coexists with the modern `gradlePlugin {}` block — delete the
  resource file. *(Picked up in the inline-fixes commit.)*
- No `gradle/libs.versions.toml`; Jackson / Kotlin / JUnit
  versions hardcoded twice in `build.gradle.kts`.
- Group/version source-of-truth duplicated across
  `pyproject.toml` + `gradle-plugin/build.gradle.kts` + READMEs.
- Min Gradle version is documented but not enforced
  (`GradleVersion.current() < ...` check missing).
- `UNCHECKED_CAST` of `target_lang_paths` in
  `TranslateBundlesTask.kt:160-162` — replace with explicit
  per-key mapping or a typed helper.
- `ERR_INTERNAL` envelope leaks `str(exc)` directly
  (`daemon.py:258`) — gate on a `--debug` flag, sanitize default.
- TM connection rebuilt per `translate_file` call
  (`daemon.py:361-378`) — cache by `tm_path` alongside the
  existing `_routers` cache.
- `redirectError(INHERIT)` in `DaemonClient.start`
  (`DaemonClient.kt:186`) — Windows pipe deadlock risk; capture
  on a pump thread.
- `test_daemon.py:259` accepts either `"internal"` or
  `"provider-failure"` for an unknown-provider error — pin one.
- Dead helper `_now_ms()` in `_usage_log.py:190-193`. *(Picked up
  in the inline-fixes commit.)*
- Premature abstraction `_build_glossary_message` in
  `anthropic_provider.py` (also openai_provider.py,
  ollama_provider.py) — kept "for cycle 3"; defer to when cycle 3
  actually wires the termbase glossary into the routing layer.
- Routing config supports `persona` / `domain` dimensions
  (`router.py:47-51`) for cycle-3+ work but no test exercises
  persona/domain matching. Add a contract test once the
  termbase / personas land.

## Inline review fixes that landed in PR #7

Tracked here so future bisects on PR #7 know which commits were
intent-driven refactors versus accidental drift:

- **#1 daemon UTF-8/newline reconfigure.** `sys.stdin` /
  `sys.stdout` reconfigured to `encoding="utf-8", newline=""`
  before the serve loop. Removes Windows CRLF-translation
  framing-corruption hazard.
- **#7 correlation-id assertion.** `DaemonClient.sendAndReceive`
  now asserts `response["id"] == callId`. Defends against any
  future interleaving / reordering at the daemon layer.
- **#10 router latency-fallback split.** The compound conditional
  `if result.latency_ms <= 0 or attributed_provider !=
  result.provider` mixed two concerns into one branch and one
  duplicate-comment block. Split into two clear `if` steps for
  bisect-friendliness; observable behavior unchanged (existing
  `test_router.py` exercises both paths).
- **`_now_ms()` deletion.** Dead helper in `_usage_log.py`,
  introduced in cycle-2 scope 2 but never wired to a caller after
  the latency-tracking design moved to `time.perf_counter()` in
  the providers themselves.
- **Legacy plugin descriptor deletion.** Removed
  `gradle-plugin/src/main/resources/META-INF/gradle-plugins/
  com.egoge.ai.nemo.translate.properties`. Redundant with the
  modern `gradlePlugin {}` block in `build.gradle.kts` —
  Gradle's plugin-publish plugin generates the descriptor from
  that block and the resource file is ignored.
